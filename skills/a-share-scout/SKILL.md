# A-Share Scout

Use this skill to build a repeatable A-share information workflow.

## Core rule

- Facts first
- Fast news second
- Theme/community clues third
- Sentiment last
- Never let creator/community content replace primary-source verification

## Five-layer source model

1. Primary factual sources
   - Exchange announcements
   - Company announcements
   - Regulatory notices
   - Company IR / investor relations pages

2. Fast market/news sources
   - 财联社
   - Other major financial breaking-news feeds

3. Theme / industry-chain clue sources
   - 韭研公社
   - Theme communities
   - Timeline / sector-expansion discussions

4. Broad verification / discovery sources
   - 东方财富
   - Quote/news/board portals with wide coverage

5. Sentiment / creator sources
   - 雪球
   - 抖音
   - Other public finance creators and discussion platforms

## What each layer is for

### 1. Facts
Best for:
- final confirmation
- original wording
- timing and scope

Not for:
- catching every hot theme early

### 2. Fast news
Best for:
- intraday catalysts
- market-moving headlines
- event timestamping

Not for:
- judging long-term theme quality by itself

### 3. Theme / industry clues
Best for:
- theme discovery
- industry-chain mapping
- expansion path and discussion intensity

Not for:
- sole factual confirmation

### 4. Broad verification
Best for:
- cross-checking names, sectors, related concepts
- quick wide scans across stocks/boards/news entrances

Not for:
- high-signal first discovery

### 5. Sentiment / creators
Best for:
- mood and narrative spread
- spotting recurring creator types
- identifying retail-facing talking points

Not for:
- sole evidence

## Creator/account evaluation

Keep accounts that usually have:
- a clear circle of competence
- recurring coverage in the same area
- traceable evidence
- clear separation of fact, inference, and opinion
- useful archives for later review

Be cautious with accounts that:
- only produce strong conclusions
- cover every hot topic equally
- rely on screenshots without sources
- maximize emotion or certainty language
- rarely revisit whether prior claims were right

## Workflow

1. Start from facts for confirmation.
2. Use fast news for new catalysts.
3. Use theme/community sources for expansion paths and industry-chain clues.
4. Use broad portals for cross-checking and horizontal search.
5. Use creator sources only for sentiment and idea flow.
6. Record recurring high-signal and low-signal accounts.
7. Separate facts, interpretations, and rumors clearly.

## Output pattern

- Source map
- Best uses by source
- Risks / noise patterns
- Candidate accounts / communities
- Minimal repeatable scan routine
- Notes on what must be verified back to primary sources
